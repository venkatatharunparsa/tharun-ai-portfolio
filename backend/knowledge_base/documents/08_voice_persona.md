## TaxSetu — The Real Story

TaxSetu didn't start as a hackathon project. It started because one of my teammate's dad got fined for a late tax filing. He was scared of taxes. He didn't understand the system. A CA charged him more than he could afford.

That bothered us. 7.34 crore MSMEs in India face this every year. Small business owners who built something real, scared of a bureaucratic system they don't understand.

So we asked — can an intelligent agent handle this completely? Not just answer questions. Actually handle it. File. Validate. Notify. Audit trail.

We built it for KSUM Hackathon in Kerala, February 2026. We were the only team from Telangana there. Judges challenged us — "is this really a problem? what are your agents actually doing?" We showed them. The system files GST autonomously with human-in-the-loop for sensitive decisions.

We didn't build it for the hackathon. We built it for the people who are scared. That difference matters.

## NINA — Why Websites Should Adapt to Humans

NINA came from frustration. Watching people struggle to fill out online forms. Watching users adapt themselves to bad interfaces.

My question was simple: why should humans adapt to websites? Websites should adapt to humans.

NINA is a voice-driven layer that sits on top of any website. You speak. It navigates. It fills forms. It executes. Zero manual interaction with complex interfaces.

The hardest part wasn't the voice. It was understanding dynamic website structures in real time. State tracking. Context preservation. That's what most voice demos skip. We didn't.

## InfraGenie — Because Infrastructure Should Be Agentic

After working with AWS I kept doing the same infrastructure tasks manually. Repetitive. Error-prone. Expensive in time.

I thought — why isn't infrastructure agentic? Why can't you say "deploy this" and have a system reason about security, cost, compliance, and execution?

InfraGenie does that. 5 agents. Hardcoded rules → RAG lookup → Gemini reasoning → human-in-the-loop for irreversible actions. The system won't let you bankrupt yourself with runaway deployments. I built the guardrails before I built the features.

That's the right order.

## About This Portfolio — Tharun AI

This portfolio itself is an agentic AI system I built to represent me.
It is not a static website — it is a live voice and text agent.

Stack I used to build it:
- LangGraph for multi-agent orchestration
- FastAPI backend deployed on Render
- Next.js frontend deployed on Vercel
- Supabase pgvector for RAG knowledge retrieval
- Groq Whisper for speech-to-text
- ElevenLabs for text-to-speech
- Upstash Redis for session memory and caching
- Gemini and Groq as LLM providers

The system has multiple specialized agents — verification, intent classification,
RAG retrieval, tool use, and contact handling — all orchestrated through LangGraph.
